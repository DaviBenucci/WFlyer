# Definition of Done

Uma tarefa só pode ser considerada concluída quando todos os critérios aplicáveis forem atendidos.

## Código

- [ ] Implementado no local correto.
- [ ] Sem duplicação desnecessária.
- [ ] Tipado quando aplicável.
- [ ] Sem secrets no código.
- [ ] Sem hacks temporários não registrados.

## Segurança

- [ ] Entradas validadas.
- [ ] Erros públicos seguros.
- [ ] Logs sem dados sensíveis.
- [ ] Upload/download seguros quando aplicável.
- [ ] Autorização/token validado quando aplicável.

## Testes

- [ ] Unitários relevantes.
- [ ] Integração quando API/banco/worker mudar.
- [ ] E2E quando fluxo de usuário mudar.
- [ ] Testes musicais quando regra de transposição mudar.
- [ ] Testes registrados no log.

## UX/Acessibilidade

- [ ] Estados loading/error/empty.
- [ ] Mobile e desktop verificados.
- [ ] Acessibilidade básica.
- [ ] Reduced motion respeitado.

## Documentação

- [ ] Docs atualizadas.
- [ ] Logs atualizados.
- [ ] Decisões registradas.
- [ ] Pendências explícitas.
